# 音楽ゲーム仕様書（v1.0）

## 1. ゲーム基本仕様

### 1.1 概要

本ゲームは、縦落ち型の4キー/6キー対応音楽ゲームであり、
ユーザー投稿を前提としたランキング競技型ゲームである。

### 1.2 プレイ形式

- 縦落ち型レーン
- 判定ライン固定
- スクラッチなし
- Tap / Hold ノート対応
- 同時押し対応（全押し含む）

### 1.3 キー数

- 4キー / 6キー両対応
- 同一楽曲に複数キー譜面を持てる

### 1.4 プレイ目的

- オンラインランキングによるスコア競争

### 1.5 方針

- プレイ体験は王道
- 独自性は投稿機能に集中

---

## 2. コンテンツパッケージ仕様

### 2.1 単位

- 1曲 = 1パッケージ
- 1パッケージは複数譜面を含む

### 2.2 ディレクトリ構造

```
Song/
├── song.json       # メタ情報
├── audio.mp3       # 音源
├── jacket.png      # ジャケット画像
└── charts/
    └── *.chart     # 譜面ファイル（複数可）
```

### 2.3 メタ情報（`SongMeta`）

`song.json` に格納する。全フィールド必須。

| フィールド            | 型        | 説明            |
|------------------|----------|---------------|
| `songId`         | `string` | 楽曲の一意識別子      |
| `title`          | `string` | 楽曲タイトル        |
| `artist`         | `string` | アーティスト名       |
| `baseBpm`        | `float`  | 基準BPM         |
| `audioFile`      | `string` | 音源ファイル名       |
| `jacketFile`     | `string` | ジャケット画像ファイル名  |
| `previewStartMs` | `int`    | 選曲画面プレビュー開始位置 |
| `songVersion`    | `int`    | パッケージバージョン    |

### 2.4 識別子体系

| 識別子       | スコープ | 用途         |
|-----------|------|------------|
| `SongId`  | 楽曲単位 | 楽曲の一意識別    |
| `ChartId` | 譜面単位 | 譜面の一意識別    |
| `Version` | 更新単位 | パッケージの世代管理 |
| `Hash`    | 内容単位 | 改ざん検知・差分管理 |

### 2.5 配布形式

- 独自拡張子（内部はzip）

### 2.6 更新方式

- 表示上は上書き更新
- 内部的には `Hash` による差分管理

## 3. 譜面仕様

### 3.1 ファイル形式

- 独自テキスト形式（CSV、1行1イベント）
- エンコード：UTF-8
- コメント行：`#` で開始

### 3.2 セクション構成

譜面ファイルは以下の3セクションで構成される。全セクション必須。

#### `[Metadata]` — 譜面メタ情報（`ChartMeta`）

| フィールド           | 型        | 説明                |
|-----------------|----------|-------------------|
| `chartId`       | `string` | 譜面の一意識別子          |
| `keyCount`      | `int`    | キー数（4 or 6）       |
| `difficulty`    | `string` | 難易度名              |
| `level`         | `int`    | 難易度レベル            |
| `chartAuthor`   | `string` | 譜面作者              |
| `formatVersion` | `int`    | フォーマットバージョン       |
| `resolution`    | `int`    | tick解像度（ファイルごと指定） |

#### `[Timing]` — タイミングイベント（`TimingEvent`）

| イベント種別  | パラメータ         | 説明    |
|---------|---------------|-------|
| `bpm`   | `tick`, `bpm` | BPM変更 |
| `meter` | `tick`, `n/d` | 拍子変更  |

#### `[Notes]` — ノートデータ（`Note`）

| ノート種別  | パラメータ                     | 説明                   |
|--------|---------------------------|----------------------|
| `tap`  | `tick`, `lane`            | 単押しノート               |
| `hold` | `tick`, `lane`, `endTick` | 長押しノート（`endTick` 必須） |

### 3.3 時間基準

| 用途 | 単位     | 説明                       |
|----|--------|--------------------------|
| 保存 | `tick` | 譜面ファイル上の記録単位             |
| 判定 | `ms`   | 実行時に `Timing` 情報から変換して使用 |

### 3.4 バリデーション制約

以下に違反する譜面はロード時にエラーとする。

- レーン範囲外のノート禁止（`0 <= lane < keyCount`）
- 負の `tick` 値禁止
- 音源長を超過する `tick` 禁止
- Hold の `endTick <= tick`（逆転）禁止
- 同一レーンへのノート重複禁止（Hold区間を含む）
- 不正な `type` 値
- 必須セクションの欠如
- 数値フィールドの型不正
- Hold における `endTick` の欠如

### 3.5 拡張性

- 未知のキーは警告のみ（エラーとしない）
- `formatVersion` により互換性を管理

## 4. 判定仕様

### 4.1 判定種別（`JudgeResult`）

`Perfect` / `Great` / `Good` / `Bad` / `Miss`

### 4.2 判定方式

- 時刻ベース（ノートの理想到達時刻と入力時刻の差分）

### 4.3 Tap判定

- キー押下時のみ判定を行う

### 4.4 Hold判定

- 始点：Tap と同様に押下タイミングで判定
- 維持：Hold中にキーを離した場合 `Miss`

### 4.5 早押し

- 判定窓外の入力は無視（ペナルティなし）

### 4.6 Miss条件

- 判定窓を未入力のまま通過
- Hold中のキー離し

### 4.7 同時押し

- 各ノートを独立に判定（相互干渉なし）

## 5. スコア仕様

### 5.1 スコア構成（`ScoreResult`）

- 判定加点方式
- コンボ補正あり

### 5.2 正規化

- 全譜面共通の固定最大スコア

### 5.3 ランキング

- 総合スコア降順

### 5.4 達成率（`AchievementRate`）

- 判定点ベース（コンボ補正なし）

### 5.5 Hold得点

- 始点判定のみ得点対象

### 5.6 リザルト表示（`ResultData`）

| 項目              | 説明             |
|-----------------|----------------|
| `score`         | 最終スコア          |
| `achievement`   | 達成率            |
| `judgeCounts`   | 判定内訳（種別ごと件数）   |
| `maxCombo`      | 最大コンボ数         |
| `avgOffset`     | 平均ズレ（ms）       |
| `fastSlowCount` | Fast / Slow 内訳 |

### 5.7 サーバー送信データ（`ScoreSubmission`）

| フィールド           | 説明             |
|-----------------|----------------|
| `score`         | スコア            |
| `judgeCounts`   | 判定内訳           |
| `maxCombo`      | 最大コンボ          |
| `options`       | 使用オプション        |
| `clientVersion` | クライアントバージョン    |
| `inputLog`      | 入力ログ / リプレイデータ |

## 6. ゲージ・クリア仕様

### 6.1 ゲージ（`Gauge`）

- 初期値：100%
- 判定による増減：

| 判定                  | 効果   |
|---------------------|------|
| `Perfect` / `Great` | 回復   |
| `Good`              | 変化なし |
| `Bad` / `Miss`      | 減少   |

### 6.2 クリア条件

- 曲終了時にゲージが閾値以上であること
- 途中失敗なし（常に完走可能）

### 6.3 ランク（`Rank`）

達成率に基づいて付与。

`SS` / `S` / `A` / `B` / `C` / `F`

### 6.4 特殊称号

| 称号           | 条件                |
|--------------|-------------------|
| `FullCombo`  | `Miss` / `Bad` が0 |
| `AllPerfect` | 全ノート `Perfect`    |

### 6.5 ランキング参加条件

- クリア（完走）必須
- `Failed` 状態では登録不可

## 7. ローカルデータ管理

### 7.1 ディレクトリ構成

- 1曲1フォルダ（セクション2.2の構造に準拠）

### 7.2 読み込み方式（`SongLoader`）

- メタ情報（`song.json`）を先読み
- 譜面データは選択時に遅延ロード

### 7.3 対応フォーマット

| 種別 | 対応形式      |
|----|-----------|
| 音源 | mp3       |
| 画像 | png / jpg |

### 7.4 エラー処理

- 読み込み失敗した楽曲はスキップ
- エラー一覧をまとめて表示

## 8. 画面構成

### 8.1 画面一覧（`Scene`）

| 画面           | 概要     |
|--------------|--------|
| `Title`      | タイトル画面 |
| `SongSelect` | 曲選択画面  |
| `Play`       | プレイ画面  |

- 難易度切替

### 8.3 プレイ画面（`PlayScene`）

| UI要素 | 説明            |
|------|---------------|
| レーン  | ノート落下領域       |

| 判定表示 | 判定結果のフィードバック  |

- ポーズ機能あり
- ポーズ使用時はランキング対象外

## 9. 入力仕様

### 9.1 キー配置（`KeyConfig`）

| モード | キー配置                    |
|-----|-------------------------|
| 4キー | `D` `F` `J` `K`         |
| 6キー | `S` `D` `F` `J` `K` `L` |

### 9.2 UI操作キー

| キー                | 操作     |
|-------------------|--------|
| `Enter`           | 決定     |
| `Esc`             | 戻る     |
| `↑` `↓` / `W` `S` | カーソル移動 |
| `R`               | リトライ   |

### 9.3 入力処理（`InputHandler`）

- 押下イベント + 状態管理方式
- OSキーリピートは無視

### 9.4 同時押し処理

- 同一フレーム内の全入力を取得
- 入力順序に非依存

### 9.5 フォーカス喪失時

- 自動ポーズ
- ランキング無効化

### 9.6 キーコンフィグ

- プレイキーとUIキーは分離管理
- 4キー / 6キー配置を個別保存
- 同一キーの重複割り当て禁止
