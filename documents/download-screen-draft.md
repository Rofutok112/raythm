# ダウンロード画面 Draft

## 目的

- `raythm-server` 上の楽曲・譜面を、ゲーム内から閲覧してダウンロードできる画面を追加する
- この branch では実装はせず、まず画面導線と構成だけを固める
- HOME 導線に自然につながることを優先する

## 前提ルール

- このゲームでは楽曲は重複しない
- すべての譜面は必ず 1 つの楽曲に属する
- したがって、ダウンロード画面の基本構造は `楽曲一覧 -> 選択中楽曲の譜面一覧` の親子構造にする
- 楽曲と譜面にはそれぞれ `公式 / 非公式` の区別がある

## 入口

- `HOME -> ONLINE`
- できるだけ hard な scene 切替感は避けて、今の hub の延長として入る

## 画面の大方針

- 現行の曲選択画面と同じ配置にはしない
- ただし、タイトル / HOME で使っている軽めの空気感、余白、半透明カード、横に広い見せ方は継承する
- 機能の主語はあくまで `楽曲` に置く
- 譜面は選択中楽曲の子要素として表示する

## カタログの分け方

`ONLINE` を開いたあと、最上段でカタログ種別を明確に分ける。

- `OFFICIAL`
- `COMMUNITY`

初期案としては「完全に別 scene に増やす」のではなく、同一 scene 内でモードを切り替える構成を推奨する。

理由:

- HOME からの導線を 1 本に保ちやすい
- 実装コストを抑えつつ「公式と非公式を混ぜない」という要件を満たせる
- 将来的に `ONLINE` 配下へ別機能を増やしても整理しやすい

見た目としては `OFFICIAL` と `COMMUNITY` を同じ一覧のフィルタに見せず、明確に別画面へ入った感触を持たせる。

## 想定スコープ

- 公開されている楽曲・譜面は、未ログインでも閲覧可能
- 楽曲一覧を見て、選択中楽曲に属する譜面を確認できる
- Jacket、曲名、アーティスト、軽めのメタ情報を表示する
- Upload / Create 系とは分けて扱う
- 文字検索を必須で入れる

## 検索の扱い

タグはまだ存在しないため、初期実装では文字検索に絞る。

- 検索対象の主語は `楽曲`
- 検索入力は画面上部の常設要素にする
- `OFFICIAL` と `COMMUNITY` は別々に検索する
- まずは以下の文字一致を想定する
  - 曲名
  - アーティスト名
  - 必要なら song id

初期段階では、譜面名や譜面作者まで横断検索する必要はない。
譜面は楽曲を選んだあとに子一覧として比較する前提に寄せる。

## ダウンロード単位

親子構造を前提に、初期実装では次の優先順位にする。

- 主ダウンロード単位は `楽曲`
- 譜面は `選択中楽曲に属する子要素`
- 譜面単体ダウンロードは残してよいが、主導線にはしない

具体的には:

- `楽曲をダウンロード` を主ボタンに置く
- 各譜面行には個別ダウンロード状態を表示する
- 楽曲が未所持でも、譜面だけ先に目立つ UI にはしない

この方針にすると、「曲は絶対重複しない」「譜面は必ず親楽曲に属する」というデータルールと画面の振る舞いが一致する。

## ローカル重複と更新

同じ楽曲 id のデータがすでにローカルにある場合、無言で上書きしない。

楽曲単位では次の状態を持てるようにする。

- 未ダウンロード
- ダウンロード中
- インストール済み
- 更新あり

譜面単位でも同様に、最低限の状態表示を持てるようにする。

- 未ダウンロード
- ダウンロード中
- インストール済み
- 更新あり

同一 id の楽曲がローカルにある場合:

- 主ボタンを `ダウンロード` のままにせず、`更新` か `再取得` に変える
- ローカルで開けることを分かるようにする
- 既存データを持っていることを、一覧または詳細で即座に分かるようにする

## レイアウト方向

構成は「左 1 本の巨大リスト + 右で詳細」よりも、HOME の流れを引き継いだ横長レイアウトに寄せる。

初期案:

- 上部:
  - `OFFICIAL / COMMUNITY` 切替
  - 検索バー
  - アカウント状態
- 左:
  - 楽曲一覧
- 中央:
  - 選択中楽曲のメイン表示
  - Jacket
  - 曲名 / アーティスト
  - メタ情報
- 右:
  - 子譜面一覧
  - ダウンロード状態
  - 行動ボタン

ただし、現行曲選択のような「管理画面っぽい固定ボックス並び」には寄せない。
各領域は太いパネルよりも、余白と淡い境界、半透明カードで分ける。

## 公式 / 非公式の見せ方

`OFFICIAL` と `COMMUNITY` は混在させない。

- `OFFICIAL`
  - curated な印象
  - デフォルトで安心して取れる楽曲群
  - 画面トーンは少し整った見せ方
- `COMMUNITY`
  - ユーザー投稿や拡張譜面寄り
  - より更新頻度や譜面差分を意識した見せ方
  - 非公式であることを明示する

同じ UI 骨格を使ってもよいが、ヘッダーテキストや補助コピーは分ける。

## 初期実装で固定してよい判断

- `ONLINE` は最初からダウンロード閲覧画面に入る
- その中で `OFFICIAL / COMMUNITY` を切り替える
- 検索は必須
- 一覧は楽曲主体
- 譜面は必ず選択中楽曲の子一覧
- 重複時は無言上書きしない

## 実装へ渡すための状態

最低限ほしい画面状態:

- 現在のカタログ種別
  - `OFFICIAL` / `COMMUNITY`
- 検索文字列
- 楽曲一覧
- 選択中楽曲
- 選択中楽曲に属する譜面一覧
- 楽曲ごとのローカル所有状態
- 譜面ごとのローカル所有状態
- 楽曲ダウンロード進行状態
- 譜面ダウンロード進行状態

## 今回あえて残すもの

今回の Draft では、以下はまだ細かく決め切らない。

- サーバー API の形
- ダウンロード失敗時のリトライ UI
- ログイン必須操作の境界
- 将来的に `ONLINE` を `Download / Multiplayer / Rankings` に分割するかどうか

まずは、

> 公式 / 非公式を分けた上で、楽曲親子構造と検索を中心にしたダウンロード画面の見た目と導線を成立させること

を優先する。
